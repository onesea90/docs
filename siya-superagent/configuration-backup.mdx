## Customize Siya to Your Needs

Siya offers extensive configuration options to tailor its behavior to your preferences and requirements. From appearance settings to advanced AI parameters, you can fine-tune every aspect of your experience.

## Accessing Configuration

### macOS Desktop App

**Main Preferences**
- Siya menu → Preferences (⌘,)
- Or use the settings icon in the toolbar

**Quick Settings**
- Right-click in chat area
- Use keyboard shortcuts
- Access from status bar

## General Settings

### Account & Subscription
- **Profile Information** - Your account details
- **Subscription Status** - Current plan and features
- **Usage Statistics** - Track your usage
- **Billing** - Manage payment methods

### Updates
- **Automatic Updates** - Enable/disable auto-update
- **Update Channel** - Stable or beta releases
- **Check Now** - Manual update check
- **Release Notes** - View what's new

## AI Model Settings

### Model Selection
- **Default Model** - Primary AI model for new chats
- **Available Models**:
  - Claude (Opus, Sonnet, Haiku)
  - OpenAI (GPT-4, GPT-4 Turbo, GPT-3.5)
  - Google (Gemini Pro, PaLM 2)
  - Local Models (Llamafile support)
- **Quick Switch** - Change models mid-conversation
- **Task-Based Selection** - Different models for different tasks

### Model Configuration
- **Temperature** - Creativity level (0.0-1.0)
- **Max Tokens** - Response length limit
- **Top P** - Nucleus sampling parameter
- **Frequency Penalty** - Reduce repetition

### API Keys
- **Anthropic API Key** - For Claude models
- **OpenAI API Key** - For GPT models
- **Google API Key** - For Gemini/PaLM
- **Custom Endpoints** - For enterprise/local models

### Local Models
- **Models Directory** - `~/.siya/local-models/`
- **Auto-Detection** - Finds llamafiles automatically
- **Performance Settings** - Thread count, memory usage
- **Model Management** - Add/remove local models
