## Welcome to the Future of AI Assistance

Siya represents a breakthrough in AI technology, featuring an adaptive multi-agent system that changes its architecture based on how you choose to work. Whether you prefer interactive collaboration or autonomous execution, Siya adjusts to deliver optimal results.

## Why Siya?

### The Challenge
Modern tasks require diverse expertise and approaches. Traditional AI assistants often struggle with complex, multi-faceted challenges that span different domains. Users need both interactive assistance for exploration and autonomous execution for well-defined tasks.

### The Solution
Siya solves this with two distinct execution modes:
- **Chat Mode**: Interactive assistance with tools and spawnable sub-agents
- **Task Mode**: Autonomous orchestration of specialized agent modules

This dual-mode architecture ensures you always have the right approach for your needs.

## Core Architecture Components

### 1. Siya Agent (Chat Mode)
The main AI agent that:
- Processes user requests interactively
- Has direct access to all tools
- Can spawn specialized sub-agents
- Manages conversation context
- Utilizes auto-compacting memory

### 2. Siya Orchestrator (Task Mode)
The coordination layer that:
- Analyzes complex requirements
- Creates execution plans
- Manages agent modules
- Monitors progress
- Delivers complete solutions

### 3. Memory System
Advanced memory management featuring:
- **Auto-Compacting**: Efficient token usage
- **Shared Memory**: Inter-agent communication
- **Context Preservation**: Maintains conversation state
- **Workspace Isolation**: Project-specific memory

### 4. MCP Integration
Model Context Protocol support enabling:
- Database connections
- API integrations
- Enterprise system access
- Custom tool development
- Extended capabilities

### 5. Hooks System
Event-driven automation allowing:
- Custom notifications
- Workflow triggers
- External integrations
- Performance monitoring
- Security auditing

## Technology Stack

### Core Technologies
- **Runtime**: Node.js with TypeScript
- **AI Models**: Multiple LLM providers (Anthropic, OpenAI, etc.)
- **Desktop**: Tauri framework for native apps
- **Memory**: Advanced token management with compression
- **Integration**: MCP protocol for extensibility

### Architecture Patterns
- **Modular Design**: Loosely coupled components
- **Event-Driven**: Hook system for automation
- **Adaptive**: Mode-based architecture switching
- **Scalable**: Parallel processing capabilities
- **Extensible**: Plugin-ready architecture
